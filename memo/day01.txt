웹 개발에 필요한 기술

 1. 프론트엔드
   - 디자이너와 함꼐 협업하여 디자인을 코드로 적용시킨다(퍼블리싱)
   - 사용자에게 보이는 프론트 영역에서 서버와의 통신을 통해 데이터로 주고받는다.
   - 여러 라이브러리, 프레임워크를 사용한다.
     (리액트, 앵귤러, 뷰, NEXT.JS, Jquery ...)
 
 2. 백엔드
   - 사용자에게 보이지 않는 서버쪽에서 데이터를 주고 받는다.
   - 서버마다 사용하는 언어와 프레임워크가 다르다.
     언어 : 자바, 파이썬, JS, php ...
     프레임워크 : 스프링, 장고, 라라벨 ... 
 3. 기본
   - 웹 개발을 할 때 기본적으로 알아야 하는 언어나 기술
   - HTML, CSS, JS, (>> 이 3가지를 웹표준이라 부름) Git ...

네트워크(통신망)
 - 두 대 이상의 컴퓨터 같은 기기들이 연결되어 통신하는 체계
인터넷(Internet)
 - 여러 네트워크들을 연결시켜 전세계를 연결한 네트워크(개 커)
웹(WWW)
 - 인터넷에서 동작하는 서비스 중 하나이다.(인터넷 없이 동작할 수 없다.)
 - 서로 정보를 공유할 수 있는 공간이다.
 - 인터넷이 기기간에 연결 된 큰 네트워크라면 웹은 
   그 네트워크 상에서 정보를 주고받을 수 있는 공간이자 방식이다.

웹 브라우저
 - 웹에 접속할 수 있느 소프트웨어

웹 페이지
 - 웹 상에 존재하며 특정 정보를 담고있는 문서
 - 일반 책의 페이지와는 다르게 HyperLink라는 것을 사용하여 다른 페이지로 빠르게 넘어갈 수 있으며 HyperLink는 HyperText에서 사용 가능하다
 - 웹 브라우저를 통해 사용자에게 보여진다
 - 웹 페이지는 HTML, CSS, JavaScript 등으로 만들어진다

 - 웹 페이지의 종류
   > 정적 페이지 
      : 회사소개 페이지처럼 누가 언제봐도 동일 한 정보를 보여주는 페이지
   > 동적 페이지
      : 우리가 일상적으로 이용하는 페이지처럼 누가 보는지, 
웹 사이트
 - 연관된 웹 페이지들의 모음

웹 어플리케이션(웹 앱)
 - 웹 사이트가 발전하면서 단순한 정적 페이지가 아닌 사용자의 요청에 따라 동적 페이지로 보여주며ㅡ 여러 서비스를 제공하게 되었다
웹 사이트가 정적인 책의 느낌이라면 웹 앱은 마치 하나의 응용 프로드램처럼 작동을 하며 시발

인터넷의 구성요소
 서버 : 서비스를 제공하는 기기
 클라이언트 : 서비스를 제공받는 기기

 웹 사이트를 접속한다는 것은 웹 브라우저를 통해 서버에 요청을 전송하느 것이며 서버에서 요청을 올바르게 처리하여 다시 클라이언ㅋ트에게 응답을 보내면 클라이언트는 웹 브라우저를 통해 데이ㅏ터를 받아보는 것이다


인터넷의 protocol(규약)
 - IP(Internet Protocol) 
   인터넷, 네트워크에 연결한 각 기기들을 구별할 수 있는 고유한 주소

 - HTTP(HyperText Transfer Protocol)
   웹에서 HTML 문서를 주고받기 위한 통신규약
   초기에는 HTML만 주고받았지만 지금은 JSON, XML등 많은 것들을 주고받음
   클라이언트와 서버가 서로 응답과 요청을 주고받는 방식이다.

웹 사이트의 구성
 - HTML : 웹 페이지의 뼈대를 구성
 - CSS : HTML에 살을 입히고 꾸며준다
 - JavaScript : 움직이고 생기고 없어지는 등의 동작, 반응을 수행

HTML
 - HyperText Markup Language
   웹 사이트의 화면을 구성하는 것들 중 뼈대의 역할을 한다
   태그를 사용하며 웹 페이지가 어떤 구조로 만들어져 있는지 알려준다

태그
  -  해당 페이지에 관련된 정보를 나타내 주는 것으로 <> 를 사용한다

HTML의 기본 구조
<html> -> 웹 문서의 시작과 끝을 나타냄

	<head>
	이 페이지에 관련된 메타정보
	</head>

	<body>
	본문 -> 실제 웹 페이지에 나타나는 내용
	</body>

</html>
